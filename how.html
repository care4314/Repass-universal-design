How to Build Universally Designed Websites

Universal Design becomes real when it shows up in our HTML, CSS, JavaScript, and testing process. These techniques are practical steps you can apply to almost any project.

Techniques for Universal Design on the web

Structure and semantics

  • Use HTML5 landmarks: header, nav, main, section, article, and footer.
  • Keep a logical heading order (one h1 per page, then h2 and h3 as needed).
  • Use real buttons and links (<button> and <a>) instead of clickable <div> elements.
  • Label form controls with <label> elements and meaningful text.
  • Use ARIA attributes only when native HTML cannot express the behavior or state.

Keyboard and navigation

  • Test every page using only the keyboard: Tab, Shift+Tab, Enter, and spacebar.
  • Provide a “Skip to main content” link so keyboard users can bypass repetitive navigation.
  • Ensure that focus styles are clearly visible; never remove outlines without replacing them.
  • Avoid keyboard traps, where focus gets stuck inside a component.

Color and contrast

  • Use a contrast checker to meet at least WCAG AA (4.5:1 for normal text).
  • Do not rely on color alone to communicate meaning; pair color with icons or text labels.
  • Check text over gradients and images to make sure it is still readable.

Text and readability

  • Use relative units like rem so users can zoom and keep layouts flexible.
  • Keep line length and spacing comfortable; avoid giant paragraphs that look like a wall of text.
  • Use plain language when possible and explain any required jargon.
  • Avoid long blocks of ALL CAPS, which are harder to read and can be mispronounced by screen readers.

Images, media, and motion

  • Provide meaningful alternative text for informative images, and use empty alt="" for purely decorative ones.
  • Add captions or transcripts for videos and audio content.
  • Respect the user’s “prefers-reduced-motion” setting and avoid essential information that depends only on animation.
  • Never autoplay audio; let users decide when to play media.

Forms and errors

  • Group related fields with <fieldset> and <legend> when appropriate.
  • Write error messages that clearly explain what went wrong and how to fix it.
  • Keep touch targets large enough for thumbs—around 44×44 pixels is a good baseline.
